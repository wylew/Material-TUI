11.1 — Core Principles
#
SEMANTIC_HTML_FIRST
Use the right element for the job. Buttons are <button>, not <div onclick>. Dialogs are <dialog>. Nav is <nav>. This gives you keyboard interaction, focus management, and screen reader support for free.
$
CLASS_API_ONLY
Never add inline styles in consumer code. All variation lives in modifier classes — .metui-btn--danger, .metui-card--elevated. This makes theming, overrides, and maintenance predictable.
!
ONE_SEED_COLOR
The entire tonal palette derives from a single --seed token via color-mix(). Never hardcode accent colors. If a color feels wrong, change the seed — not individual rules.
^
NEVER_COLOR_ALONE
Never rely solely on color to convey state. Always pair with a textual indicator: [OK], [ER], [WR]. This satisfies WCAG 1.4.1 and makes logs readable in monochrome.

11.5 — Accessibility Checklist (click to check off)
All interactive elements are <button> or <a> — never <div onclick>
Every interactive element has a :focus-visible outline using --seed color
All touch targets meet the 48px MD3 minimum via min-height: var(--touch-target)
State is never conveyed by color alone — textual indicators like [OK], [ER], [WR] accompany all badges
Dialogs use native <dialog> element for free focus trapping and Escape key handling
Live regions use aria-live="polite" for log streams, toasts, and status updates
Form inputs have associated <label> elements — never placeholder-only labeling
Tab order follows visual reading order — no tabindex values greater than 0
Contrast ratio meets 4.5:1 minimum for body text against all surface backgrounds
Decorative elements (scanlines, glyphs) have aria-hidden="true" or pointer-events:none
11.6 — Do / Don't Patterns
[OK] DO — semantic state badges
[OK] SUCCESS [ER] FAILED
Text + color. Works in print, monochrome, and for colorblind users.
[ER] DON'T — color-only state
Color-only. Meaningless without vision, in monochrome, or in a log file.
[OK] DO — labelled inputs
Label is associated via for/id. Always visible, never disappears on focus.
[ER] DON'T — placeholder-only
No label. Placeholder vanishes on focus. Screen readers may not announce it.
